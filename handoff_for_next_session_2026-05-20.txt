    pre-existing render prop mismatch (`payload` vs `{bkmIds,x,y}`), broken
    since 2026-05-14, surfaced by the P6h diff review.
  • G.2.3 is DONE — all four cluster bundles are final. Do not "convert
    Cluster A" — vendor + corpus stays classic-script forever, by design.
  • STRICT-MODE TRAP: esbuild ES modules are implicitly strict. Module-private
    mutable state reassigned inside a function needs an explicit module-scope
    `let`/`const` binding or the assignment throws ReferenceError. P6+ hooks:
    prefer bare `let`/`const` over `export let` for truly module-private state.
  • COMPOSITION-LEVEL SINK PATTERN (PLAN.txt §P6 DIRECTION 4): when a fused
    effect has two concerns with different dep surfaces, split it — the
    cross-cutting sink stays in App() until all its deps are hook returns.
    P6g already applied it; P6k completes it (the persist sink).
  • WINDOW-BRIDGE CLEANUP (invariant 6): any hook that wires `window.__openX`
    MUST return `delete window.__openX` cleanup. Missing cleanup is a leak.
